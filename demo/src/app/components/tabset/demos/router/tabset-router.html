In order to easily synchronize the <ngb-tabset> with the Angular router, some directives are provided in the NgbTabsetRoutingModule module, which is not in the main entry point of ng-bootstrap (to avoid adding a general dependency on the Angular router).

The NgbTabsetRoutingModule module has to be imported from '@ng-bootstrap/ng-bootstrap/tabset/routing'.

All child <ngb-tab> elements must be linked with a route using the standard routerLink directive.

The corresponding routes have to be defined in the configuration of the Angular router.

The <ngb-tabset> should only be used on those configured routes. If no child <ngb-tab> element matches, the <ngb-tabset> will not change the active tab (and if there is no active tab, the default behavior of the normal <ngb-tabset> component is to select the first tab).

The content of the tabs can be defined either in the corresponding <ng-template ngbTabContent> elements, or as the component property defined in the router configuration (if the <router-outlet> element is included in the page).
